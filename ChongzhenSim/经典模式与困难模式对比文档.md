## 3. 功能行为影响

### 3.1 启动与模式
- 影响：`config.json` 增加 `gameplayMode`，默认改为 `classic`。
- 结果：首启由玩家选择 `classic/rigid_v1`，并按模式加载独立存档。
- 兼容：保留模式判断，后续可切回旧模式（配置层可控）。

### 3.2 状态模型
- 影响：`state.js` 新增 `mode` 与 `rigid` 子状态树。
- 结果：原状态未移除；新增刚性维度（财政/军事/朝廷/圣躬/其他、记忆锚点、事件队列等）。
- 风险：存档体积增加（`memoryAnchors` 累积，已做截断上限）。

### 3.3 回合主循环
- 影响：`turnSystem.js` 增加 rigid 分支执行器。
- 结果：在 rigid 模式下执行“校验→折扣/封驳→反弹→阈值触发→8模块→推进3个月→存档锚点”闭环。
- 新增：
  - 六科封驳“每回合至少 1 次”刚性兜底；
  - 罢朝分级（1/2/3）与复朝处置；
  - 困难专属结算入口复用统一差分面板。
- 兼容：非 rigid 模式仍走原逻辑分支。

### 3.4 叙事与交互
- 影响：`storySystem.js` 增加 rigid 渲染分支与提交锁。
- 结果：
  - 玩家侧继续看到自然叙事，不直接暴露内部模块；
  - 系统侧保留 `rigidMeta` 结构化数据，用于测试、回放与 LLM 上下文；
  - 罢朝时默认禁用施政，但允许“复朝处置”应急选项；
  - 历史分支与暗杀后续为强制处理门控；
  - 防重复提交（按钮锁）。

### 3.5 文案与体验
- 影响：`moduleComposer.js` 与 `rigidHistoryEvents.json` 深化历史叙事。
- 结果：
  - 10历史节点按 1627-1644 时序细化；
  - 模块2固定8项叙事子段；
  - 模块3固定六部+厂卫+军情，并强制输出“信息茧房矛盾”；
  - 引入“悲剧脉冲”动态文案（随阻力/焦虑/暗杀风险变化）。
  - LLM 请求体增加 `latestMemoryAnchor` 与 `lastOutputModules`，帮助困难叙事稳定贴近系统状态。

### 3.6 移动端可读性
- 影响：`theme.css` 增加小屏样式兜底。
- 结果：诏书文本和决策按钮在小屏下可读性更好。
- 备注：仍建议真机验收（仅完成样式级兜底，不含设备实测记录）。

## 8. 经典模式链路总览（玩法 / UI / 模块）

### 8.1 玩法主链路（Classic）
- 经典模式本质是“默认主链路”：除 `isRigidMode(state)` 命中时走刚性分支外，均走经典分支。
- 启动加载入口：`js/main.js`
  - `preloadBasicData(preferredMode)` 统一装载 config、characters、factions、goals、nationInit、positions，并初始化 core gameplay 状态。
  - `selectedMode = current.mode || preferredMode || config.gameplayMode || "classic"`。
  - `bootstrap()` 内按是否 `gameStarted` 进入 start 或 edict。
- 回合入口：`js/ui/edictView.js`
  - `runCurrentTurn(container, { renderId })` 进入 `js/systems/turnSystem.js`。
- 经典回合推进：`js/systems/turnSystem.js` -> `handleChoice()`（非 rigid 分支）
  1. 解析并归一化本回合 effects（含自拟诏书关键词估算、任免文本推断）。
  2. `pushCurrentTurnToHistory` 写入历史。
  3. 推进时间（月/年/日）。
  4. `processCoreGameplayTurn` 执行核心玩法结算。
  5. `resolveHostileForcesAfterChoice` 执行敌对势力后处理。
  6. 推进科举/武举与自然死亡。
  7. 季度月触发 `computeQuarterlyEffects` 并生成季度结算快照。
  8. `refreshQuarterAgendaByState` 刷新季度议题。
  9. 构建显示差分 `buildOutcomeDisplayDelta`，自动保存并重渲染。

### 8.2 剧情驱动链路（Classic）
- 渲染入口：`js/systems/storySystem.js` -> `renderStoryTurn(state, container, onChoice, options)`。
- 当前回合数据加载：`loadStoryData()`
  - 条件：`storyMode === "llm" && apiBase 非空 && 非首回合` 时请求 LLM。
  - LLM 请求：`requestStoryTurn()`（`js/api/llmStory.js`）。
  - 失败回退：本地 `data/story/*.json` 模板。
- 页面结构：
  - 历史回合块 `renderStoryHistory`。
  - 当前回合块 `renderCurrentTurn`。
  - 季度奏折面板 `renderQuarterAgendaPanel`。
  - 选项按钮 + 自拟诏书弹层 + 数值变化卡片 + 标注系统。

### 8.3 UI 结构映射
- 路由与导航：`js/router.js`
  - View：`start / edict / court / nation / settings`。
  - 切换时保存 edict 滚动位置（`saveEdictScrollTop`）。
- 顶栏与底栏：`js/layout.js`
  - `initLayout` 创建 topbar/bottombar。
  - `updateTopbarByState` 显示“崇祯X年·X月·时段·天气”。
  - `updateGoalBar` 与 `updateMinisterTabBadge` 联动目标/未读。
- 诏书页：`js/ui/edictView.js`
  - 负责触发回合渲染，不承载具体业务规则。
- 朝堂页：`js/ui/courtView.js`
  - 官员列表、聊天、忠诚标记、任命弹窗。
  - 科举/武举弹层（人才池、放榜、录用）。
  - 对话后调用统一差分卡展示“本轮对话数值变化”。
- 国家页：`js/ui/nationView.js`
  - 国家指标、朝局指标、派系支持。
  - 季度奏折、能力加点、国策树解锁、自定义国策。
  - 省份概况、敌对势力、天下大事、民间舆论。

### 8.4 核心模块职责拆分
- 状态与存档
  - `js/state.js`：全局状态树与派生 ministers 逻辑。
  - `js/storage.js`：按模式分离存档键（classic/rigid）。
- 核心玩法
  - `js/systems/coreGameplaySystem.js`：
    - 执行率、国策树、能力加点。
    - 自定义国策提取与季度加成。
    - 季度议题生成、敌对势力结算。
    - 主回合聚合 `processCoreGameplayTurn`。
- 剧情与 API
  - `js/api/requestContext.js`：组装 story/chat 请求体。
  - `js/api/llmStory.js`：剧情请求、容错解析、header 对齐。
  - `js/api/ministerChat.js`：朝臣对话请求。
  - `js/api/validators.js`：effects 限幅与 payload 归一化。
- 数值显示与效果应用
  - `js/utils/effectsProcessor.js`：nation/loyalty 数值写入。
  - `js/utils/displayStateMetrics.js`：
    - 指标定义、快照、差分、合并与卡片渲染。
    - 诏书页、朝堂页、国家页共享。

### 8.5 配置与数据驱动

### 8.6 经典模式关键文件清单（按层）

- 玩法层：`js/systems/turnSystem.js`、`js/systems/storySystem.js`、`js/systems/coreGameplaySystem.js`

### 8.7 一句话结论
